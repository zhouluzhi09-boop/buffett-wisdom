# 巴菲特如是说 (Buffett Wisdom)

> 把巴菲特八十年的投资认知，压缩成一份 AI 可直接执行的决策引擎。
>
> 这不是一个提示词。这是从 67 份原始素材中，经过 13 批次系统蒸馏产出的可安装 Claude Code Skill。

---

## Why

**你不需要另一个股票推荐工具。你需要一面镜子。**

在做投资决定之前，你希望知道——巴菲特会怎么审视这个问题？不是他 1977 年说过的某句话，而是他的**整个认知框架**——先看什么、后看什么、什么时候放弃、什么时候下重注。

这个 Skill 不告诉你答案。它展示一种你无法在别处获得的思考过程。

### 三个核心差异化

| | 普通投资 Prompt | RAG / 向量检索 | 我们这个 |
|---|---|---|---|
| 能说什么 | 巴菲特的十句名言 | 巴菲特在 X 页说过 Y | **巴菲特会怎么想** |
| 遇到新场景 | 无法推演 | 检索不到原话就沉默 | **用框架推演** |
| 能否被修正 | 改不了 | 加文档 | **L3 演化层开放给社区** |

**不是"搜索巴菲特说过什么"**。向量数据库能完美召回"安全边际 = 内在价值 - 价格"这句话，但它不知道**在四关决策流程中，安全边际是最后一个检查点，前面还有能力圈、护城河、管理层三道关**。后者是认知框架，不是句子匹配。

**不是"又一个投资 GPT"**。通用模型能写出一段像模像样的巴菲特式回答，但它不会区分 L5（绝对原则）和 L3（有依据但不绝对），不会在 L1 高频说"我不知道"，不会在每句话里藏一个长达五十年的概念演化史。

## 设计理念

### 为什么三层而不是平面知识库

```
L1 蒸馏：   巴菲特说过什么？       → 概念卡片（出处+规则+案例+边界）
L2 认知：   他怎么想的？           → 决策流程 + 优先级排序 + 排除模式
L3 演化：   这个框架有什么盲区？    → 边界声明 + L3种子问题 + 社区修正接口
```

平面知识库只能回答"已知问题"。三层架构让 Skill 能推演**新场景**（L2）和**承认盲区**（L3）。

### 7 个场景——不多不少

场景模板是用户发现 Skill 能力的地图。7 个——覆盖主航道，舍弃边缘。每多一个场景，用户的记忆负担就加一成。每少一个，就有一个高频使用模式没被覆盖。

### 五级确定性——不是概率模型

巴菲特从不说"我认为有 80% 的概率"。"模糊的正确胜过精确的错误"恰恰反对给不确定性赋概率值。五级确定性是**定性的、可沟通的**分级——L5（绝不）到 L1（我不知道），每一级都对应巴菲特真实的使用频率。

### /crazy 模式——两种人格，一套原则

年报巴菲特厚重谨慎，CNBC 巴菲特犀利直接。两种人格共享同一套核心原则、决策流程和边界约束。一个 Skill 带模式切换——不是两个 Skill——最小维护成本。

### 校验脚本——不是测试文档，是自动判卷

26 个测试用例，3 份 Python 脚本。改完 SKILL.md 跑 `/validate`，三秒知道通过率。没有脚本的测试是"打开聊天窗口一条条肉眼判"——没有人坚持做。有脚本的测试是"跑一行命令"——你会做。这两者的差距决定了项目是活的还是死的。

## 核心能力：7 类场景

| 场景 | 你问什么 | 你得到什么 |
|:--:|------|------|
| 1. 公司初判 | "帮我看看XX公司" | 四关速查表：能力圈→护城河→管理层→价格 |
| 2. 市场恐慌/狂热 | "大跌了怎么办" | 历史参照 + 市场先生分析 + 行动框架 |
| 3. 商业决策 | "该收购还是回购" | 资本配置五级优先级 + 惯性驱使检查 |
| 4. 管理层判断 | "这个 CEO 靠谱吗" | 嫁女儿测试 + 四个红旗信号扫描 |
| 5. 估值判断 | "现在贵不贵" | DCF 三问 + 安全边际方向性判断 |
| 6. 投资哲学/人生 | "怎么选职业" | 能力圈思维 + 长期复利 + 巴菲特自己的故事 |
| 7. 框架极限 | "比特币能用这方法吗" | 诚实声明 + 为什么不在框架内 |

## 使用

直接说话——不需要命令前缀：

```
巴菲特怎么看价值投资？
市场大跌了，我好慌
/crazy 这估值疯了
/case 可口可乐
```

| 命令 | 做什么 |
|------|--------|
| `/crazy 问题` | 犀利模式——CNBC 专访巴菲特 |
| `/case 公司名` | 查巴菲特历史上对该公司的经典分析 |
| `/buffett 问题` | 显式触发 Skill |
